    ESTRUCTURAS

    Las estructuras de datos en Java son formas específicas de organizar y almacenar datos para facilitar su acceso y modificación eficiente. Java ofrece una amplia variedad de estructuras que se adaptan a diferentes necesidades y tipos de problemas. A continuación, se describen las principales estructuras de datos en Java:


    Estructuras de Datos Lineales o Secuenciales

    Arrays (Arreglos):Son colecciones de elementos del mismo tipo almacenados en posiciones contiguas de memoria. Tienen tamaño fijo y permiten acceso rápido mediante índices. Útiles cuando se conoce el número de elementos y se requiere acceso directo. ArrayList: Similar a un array, pero con tamaño dinámico que puede crecer o reducirse durante la ejecución. Implementa la interfaz List y permite operaciones flexibles sobre la colección. Listas enlazadas: Consisten en nodos donde cada nodo contiene un dato y una referencia al siguiente nodo. Son dinámicas y eficientes para inserciones y eliminaciones frecuentes, especialmente en el medio o inicio de la lista. Listas doblemente enlazadas: Cada nodo tiene referencias tanto al nodo siguiente como al anterior, permitiendo navegación en ambas direcciones. Son útiles para aplicaciones que requieren retroceso, como funcionalidades de "deshacer". Colas (Queues): Siguen el principio FIFO (primero en entrar, primero en salir). Se utilizan para manejar tareas en orden de llegada, como en sistemas de impresión o procesamiento de mensajes. Pilas (Stacks): Siguen el principio LIFO (último en entrar, primero en salir). Son útiles para manejo de llamadas a funciones, evaluación de expresiones o funcionalidades de "deshacer".


    Estructuras de Datos Jerárquicas

    Árboles: Estructuras con nodos organizados en jerarquías, donde cada nodo puede tener varios hijos. Árboles binarios: Cada nodo tiene máximo dos hijos. Árboles binarios de búsqueda: Facilitan búsquedas, inserciones y eliminaciones eficientes manteniendo los datos ordenados. Otros tipos incluyen árboles rojo-negro, árboles B y árboles de sintaxis abstracta.


    Estructuras de Datos Asociativas

    Mapas (Map): Almacenan datos en pares clave-valor, permitiendo acceso rápido mediante claves únicas. Ejemplos en Java incluyen HashMap, que usa una función hash para acceso eficiente, y TreeMap, que mantiene las claves ordenadas.


    Estructuras de Datos para Grafos

    Grafos: Representan relaciones entre elementos mediante nodos (vértices) y conexiones (aristas). Aunque Java no proporciona una implementación directa, pueden construirse usando listas de adyacencia o mapas. Son esenciales en aplicaciones como redes sociales, sistemas de navegación y algoritmos de recomendación.
